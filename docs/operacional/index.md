# Time Operacional — Cadencia

Manual operacional canônico do Time Operacional. **Fonte de verdade** — o `pd-framework/times/operacional/foundation/` é cópia derivada.

## Quem é

Time ainda **não formalmente bootstrappado** (sem CLAUDE.md / persona canônica). Os documentos aqui são fundamentos antecipados que servem como playbook operacional enquanto o Time não é constituído formalmente.

## Escopo

RH, cultura, metas operacionais, processos internos, continuidade organizacional (single-point-of-failure, acessos), documentação viva e aprendizado institucional. Cobre o que Mochary chama de "areas of responsibility + no single point of failure + wiki viva + KPIs" — quase a especificação literal do mandato.

## Princípio diretor

Todo processo e decisão de estrutura passa pelo **AI Founder Mode (AFM)** — `_core/AFM.md` do framework. Contratação humana **não é o default**: é o último recurso após esgotar skill, worker, persona e refundação de processo.

## Regra de consulta obrigatória

Antes de qualquer decisão operacional na Cadencia:

| Decisão | Leia primeiro |
|---|---|
| Contratar humano (qualquer cargo) | [rh-contratacao-afm](rh-contratacao-afm.md) — sem exceção |
| Definir cargo / persona nova | [rh-contratacao-afm](rh-contratacao-afm.md) + `_core/PERSONAS.md` |
| Estruturar processo interno | `_core/AFM.md` (princípio diretor) |
| Desenhar sub-áreas ou rituais do Time | [leituras-inspiracao-desenho-time](leituras-inspiracao-desenho-time.md) |
